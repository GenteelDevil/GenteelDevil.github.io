377 组合总数IV

题目链接

思路

明确问题是完全背包+排列问题

  1. 定义dp[i]：凑成目标和为i的排列总数
  2. 确定递推公式：dp[i] += dp[i - nums[j]]
  3. dp数组初始化：dp[0] = 1，其余初始化为0
  4. 确定遍历顺序：完全背包+排列问题，因此遍历顺序：
    • 外层for循环遍历背包容量，背包容量从小到大
    • 内层for循环遍历物品
  5. 手动进行一次演算，确保给出的例子可以手动跑通

注意

  • 目前有点理解排列为什么要先遍历背包容量了，其实就是对于某个指定的背包容量，一定要把所有物品都尝试放一次，这就是排列的意义
  • 对于组合，为什么要先遍历物品，就是对于某个指定物品，将所有的背包容量全部放一次，这样物品的先后顺序就没有意义，就是组合的意义
  • dp[i] < INT_MAX - dp[i - nums[j]]

518 零钱兑换

题目

思路

分析题意，首先确认是一个完全背包，而且要求的是不同硬币方法的组合数，和排列数区分开，也就是：凑成总额为5的选择有{2, 2, 1} 和{1, 2, 2}实际上都是一样的，算一种方法，而排列数算为两种方法

  1. 定义dp[j]：凑成总金额为j的货币组合数为dp[j]

  2. 递推公式：dp[j] = dp[j] + dp[j - nums[i]]

  3. 初始化：dp[0] = 1，理解为凑到金额0只有全部硬币都不取一种方法

  4. 确认遍历顺序

    如果是对于01背包，没有什么选择，先遍历物品，再遍历背包容量，且背包容量是从大到小

    但是对于完全背包，可以确认的是背包容量是从小遍历到大，这样可以保证物品的重复放入属性，需要根据不同的需要判定物品和背包容量的遍历先后顺序

    如果是对于求组合数，也就是这一道题，需要先遍历物品，再遍历背包容量；

    如果是求排列数，那么需要先遍历背包容量，再遍历物品；

注意

  • 求组合，先遍历物品，再遍历背包容量；求排列，先变量背包容量，再遍历物品
  • 基本上明白dp该怎么进行定义了，题目怎么需要就怎么定义，无非是方法数（排列、组合）、最值，先就这两种吧

12 矩阵中的路径

发表于 2021-06-21 分类于 算法 ， 剑指Offer

12 矩阵中的路径

题目

给定一个m x n二维字符网格board和一个字符串单词word。如果word存在于网格中，返回 true ；否则，返回 false 。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

思路

这道题使用的递归，递归的思路：

  • 递归参数

    当前元素在矩阵中的行和列索引、已经匹配的长度k

  • 中止条件

    • 返回false：（1）当前索引越界 （2）当前矩阵的元素与目标字符不一致（3）矩阵元素已经访问过
    • 返回true：k = len(word) - 1，说明已经全部匹配

  • 递推工作

    1. 标记当前矩阵元素：board\[i][j] = ' '，代表已经访问过，防止以后的重复搜索

    2. 搜索下一单元格：上下左右四个方向开启递归，并记录结果

    3. 还原当前的矩阵：将board[i][j]还原到初始值，也就是word[k]

      Cm：递归初始参数的设定，因为是已经匹配了长度k，那么word[k]就是当前匹配成功的元素

  • 返回值

    返回bool变量res，代表是否搜索成功

注意

  • 函数传参，该引用的时候引用，可以极大提高空间和时间复杂度
  • 明确递归函数的定义和变量的定义

复杂度分析

  • 时间复杂度：O(3^KMN)，最差的情况就是遍历所有的元素一共MN个，每一个都匹配K次，每次有三个方向的选择
  • 空间复杂度：O(K)

基础概念说明

在这个模块封装了线程同步的机制：POSIX信号量、互斥量和条件变量，对于三者的API简单介绍可以参见线程间同步的方式
