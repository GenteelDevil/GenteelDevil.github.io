518 零钱兑换

题目

思路

分析题意，首先确认是一个完全背包，而且要求的是不同硬币方法的组合数，和排列数区分开，也就是：凑成总额为5的选择有{2, 2, 1} 和{1, 2, 2}实际上都是一样的，算一种方法，而排列数算为两种方法

  1. 定义dp[j]：凑成总金额为j的货币组合数为dp[j]

  2. 递推公式：dp[j] = dp[j] + dp[j - nums[i]]

  3. 初始化：dp[0] = 1

  4. 确认遍历顺序

    如果是对于01背包，没有什么选择，先遍历物品，再遍历背包容量，且背包容量是从大到小

    但是对于完全背包，可以确认的是背包容量是从小遍历到大，这样可以保证物品的重复放入属性，需要根据不同的需要判定物品和背包容量的遍历先后顺序

    如果是对于求组合数，也就是这一道题，需要先遍历物品，再遍历背包容量；

    如果是求排列数，那么需要先遍历背包容量，再遍历物品；

注意

  • 求组合，先遍历物品，再遍历背包容量；求排列，先变量背包容量，再遍历物品
  • 基本上明白dp该怎么进行定义了，题目怎么需要就怎么定义，无非是方法数（排列、组合）、最值，先就这两种吧

12 矩阵中的路径

题目

给定一个m x n二维字符网格board和一个字符串单词word。如果word存在于网格中，返回 true ；否则，返回 false 。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

思路

这道题使用的递归，递归的思路：

  • 递归参数

    当前元素在矩阵中的行和列索引、已经匹配的长度k

  • 中止条件

    • 返回false：（1）当前索引越界 （2）当前矩阵的元素与目标字符不一致（3）矩阵元素已经访问过
    • 返回true：k = len(word) - 1，说明已经全部匹配

  • 递推工作

    1. 标记当前矩阵元素：board\[i][j] = ' '，代表已经访问过，防止以后的重复搜索

    2. 搜索下一单元格：上下左右四个方向开启递归，并记录结果

    3. 还原当前的矩阵：将board[i][j]还原到初始值，也就是word[k]

      Cm：递归初始参数的设定，因为是已经匹配了长度k，那么word[k]就是当前匹配成功的元素

  • 返回值

    返回bool变量res，代表是否搜索成功

注意

  • 函数传参，该引用的时候引用，可以极大提高空间和时间复杂度
  • 明确递归函数的定义和变量的定义

复杂度分析

  • 时间复杂度：O(3^KMN)，最差的情况就是遍历所有的元素一共MN个，每一个都匹配K次，每次有三个方向的选择
  • 空间复杂度：O(K)

线程间的同步的方式

线程间同步的方式主要有三种：信号量、互斥量（互斥锁）和条件变量

  • 信号量类似于多进程的信号量
  • 互斥量，或者说互斥锁，保护关键代码段，以确保线程的独占式访问
  • 条件变量，用于线程之间同步共享数据的值，也提供了一种通知机制，当某个共享数据达到某个值的时候，唤醒等待这个共享数据的线程

POSIX信号量

POSIX信号量和进程间通信的System V IPC信号量是不同的，接口比较相似

常用的信号量操作方式：初始化信号量、销毁信号量、PV操作之类，如以下API

初始化和销毁信号量都要小心，防止初始化已经初始化过的信号量、防止销毁正在有进程使用的信号量

互斥量

或者说，互斥锁，用来保护关键代码段，以确保独占式的访问，类似于二进制信号量。

对于互斥量，也可以设置是否允许进程间共享、只能一个进程的多个线程内共享

条件变量

用于线程之间共享同步数据的量，当共享数据达到某个值的时候，唤醒这个等待共享数据的线程

一个程序是从main函数开始执行么

发表于 2021-06-18 分类于 操作系统

一个程序是从main函数开始执行么

不是的

  • 一个程序开始运行，实际上是操作系统为它创建了一个进程后，把控制权交到了入口函数Entry Point，入口函数也往往是运行库的一部分。
  • 入口函数对运行库和程序运行环境进行初始化，包括堆栈、IO、线程、全局变量的构造等等。
  • 入口函数初始化之后，调用main函数，并且传入main函数需要的参数，argc、argv字符串数组
  • main函数执行完毕之后，实际上就是函数调用完毕，返回到入口函数，入口函数进行清理工作，例如全局变量的析构、堆销毁、关闭IO等，然后进行系统调用结束进程

额外的一些补充，glibc的入口是_start，我们也可以在编译的时候来重写_start函数，重写了之后原来_start给main函数准备的退出清理工作就没了，这个时候main函数的退出也不能使用return，而是要使用exit。同样编译的使用也可以使用-e属性，或者是-nostartfiles，具体就不展开说了，参考链接

_start函数会调用__libc_start_main函数，大概的基本流程如下，可以总结为，获取参数列表和环境变量，为下一步调用准备：

然后执行函数__libc_start_main，这个时候还没有执行到main函数，不过在这个里面要进行调用了

exit函数也是有讲究的，不过根本就是使用系统调用结束进程
